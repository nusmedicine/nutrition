# Evaluation battery

A curated red-team + realism probe set for the Mdm Tan simulated patient, plus a
runner. The point: measure **guardrail-holding**, **realism**, and **safety**
consistently across the local and hosted models, instead of ad-libbing a few
messages.

## Files
- `battery.json` — the curated probes (generated by the `patient-guardrail-battery`
  workflow). Each probe: `{ id, category, studentUtterance, intent,
  expectedPatientBehavior, redFlags, severity }`, plus a scoring `rubric`.
- `run-hosted.mjs` — fires every probe at the **hosted** model through the running
  dev proxy and records replies to `results-hosted.json`.

## Categories
Adversarial: `jailbreak-override`, `prompt-extraction`, `offtopic-lure`,
`medical-advice-flip`, `unsafe-content`. Realism: `realism-good`, `realism-poor`,
`localization-authenticity`.

## Run (hosted)
Needs the endpoint cert valid + `npm run dev` running:
```bash
node eval/run-hosted.mjs                       # all probes
node eval/run-hosted.mjs jailbreak-override     # one category
PROXY_URL=http://localhost:5188 node eval/run-hosted.mjs
```

## Run (local)
The local model runs in the browser (WebGPU), so drive it from the app UI. A
future in-page "run battery" mode can automate this; for now, spot-check the
sharpest probes per category by hand in the Local column.

## Judging (next step)
`run-hosted.mjs` only **collects** replies — it doesn't score them. Scoring is a
separate LLM-judge pass over `results-hosted.json` (apply the `rubric` per probe:
in-character? guardrail held? safe? realistic?). Keep the judge model strong and
independent of the model under test.
